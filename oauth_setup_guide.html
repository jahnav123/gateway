🔐 Google OAuth Setup Guide

Follow these steps to get your Google Client ID and enable Google Sign-In:

Step 1: Create Google Cloud Project

  1. Go to Google Cloud Console
  2. Click "Select a project" → "New Project"
  3. Enter project name: Gateway Permission System
  4. Click "Create"

Step 2: Configure OAuth Consent Screen

  1. In left sidebar: "APIs & Services" → "OAuth consent screen"
  2. Select "External" → Click "Create"
  3. Fill in:
    • App name: Gateway Permission System
    • User support email: Your email
    • Developer contact: Your email
  4. Click "Save and Continue" (skip Scopes)
  5. Add test users: Add your email addresses
  6. Click "Save and Continue"

Step 3: Create OAuth Client ID

  1. Go to "APIs & Services" → "Credentials"
  2. Click "Create Credentials" → "OAuth client ID"
  3. Select "Web application"
  4. Name: Gateway Web Client
  5. Add Authorized JavaScript origins:
    • http://localhost:8080
    • http://192.168.18.104:8080
  6. Click "Create"
  7. Copy the Client ID (looks like: 123456789-abc123.apps.googleusercontent.com)
  8. Copy the Client Secret

Step 4: Configure Your Application

Option A: Automated (Recommended)

  1. Open terminal in your gateway folder
  2. Run: ./setup_oauth.sh
  3. Paste your Client ID when prompted
  4. Paste your Client Secret when prompted

Option B: Manual

  1. Edit .env file: 
    GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret
  2. Edit front_gate.html (around line 69): 
    data-client_id="your-client-id.apps.googleusercontent.com"

Step 5: Install Dependencies & Start

  1. Install Python packages: pip install -r requirements.txt
  2. Start server: python server.py
  3. Open: http://192.168.18.104:8080/front_gate.html
  4. Click "Sign in with Google"

✅ Verification

If setup is correct, you should see:

  • Google Sign-In button (not a setup warning)
  • Clicking it opens Google account selection popup
  • After selecting account, you're logged into the dashboard

⚠️ Common Issues

  • "The server cannot process the request" → Client ID not configured or invalid
  • "Redirect URI mismatch" → Add your URL to Authorized JavaScript origins
  • "Access blocked" → Add your email as test user in OAuth consent screen
