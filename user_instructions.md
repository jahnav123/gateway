# Student Gateway Permission System - User Instructions

## System Objective
Digitally manage student off-campus permission requests through a multi-level approval workflow, eliminating paper-based processes and reducing faculty interruptions.

## How to Use

### For Students
1. **Login**: Access the system with your roll number and password
2. **Dashboard Menu**: After login, access the menu with the following tabs:
   - **Pending**: View all requests awaiting approval (parent, teacher, or HOD)
   - **Approved**: View all approved requests with leave details
   - **Denied**: View all rejected requests with rejection reasons
   - **Cancelled**: View all requests you have cancelled
   - **All Requests**: View complete history of all submitted requests
   - Click on any request to see complete details including approval chain and timestamps
3. **Submit Request**: 
   - Select request type (Emergency or Casual)
   - Enter reason for leaving (max 500 characters)
   - Choose leave date and time
   - Submit request
4. **Cancel Request**: You can cancel any pending request before final HOD approval from your dashboard
   - Available for requests in Pending tab only
   - Notifications sent to parent/teacher/HOD if already in their queue
5. **Receive Notification**: Get SMS confirmation when request is approved with leave details
6. **Show Approval**: Present approval SMS at gate during leave time

### For Parents
1. **Receive SMS**: Get notification with request details and approval link
2. **Review Request**: Click link to view student's permission request details
3. **Take Action**: Click "Approve" or "Reject" button
4. **Confirmation**: Receive SMS confirming your action

### For Class Teachers
1. **Login**: Access teacher dashboard with your credentials
2. **View Pending**: See all parent-approved requests from your classes
3. **Review Details**: Click on request to view complete information including parent approval
4. **Approve/Reject**: Take action on requests
5. **Batch Processing**: Review multiple requests during your free time without classroom interruptions

### For HOD (Head of Department)
1. **Login**: Access HOD dashboard with your credentials
2. **Department Overview**: View all pending requests across your department
3. **Filter Requests**: Sort by class, request type, or date range
4. **Review Chain**: See complete approval history (parent + teacher approvals)
5. **Final Decision**: Approve or reject requests
6. **Analytics**: Monitor department statistics and approval patterns

## System Features
- **Real-time Status**: Track request progress through approval chain
- **SMS Notifications**: Automatic alerts to parents and students at each stage
- **Auto-Expiry**: Approved requests automatically expire after the specified leave time
- **Audit Trail**: Complete record of all approvals with timestamps
- **Mobile-Friendly**: Access from any device, optimized for mobile phones
- **No Duplicates**: System prevents multiple pending requests from same student for permission on same day

## Request Status Indicators
- **Pending Parent**: Awaiting parent approval via SMS
- **Pending Teacher**: Parent approved, awaiting class teacher review
- **Pending HOD**: Teacher approved, awaiting HOD final approval
- **Approved**: Fully approved, valid until leave time expires
- **Rejected**: Denied by parent, teacher, or HOD
- **Cancelled**: Cancelled by student before final approval
- **Expired**: Leave time has passed (appears only in All Requests tab)

## Tips
- **Students**: Provide clear, specific reasons for leave requests
- **Parents**: Respond to SMS links within 24 hours (links expire)
- **Teachers**: Review requests during designated free time to avoid interruptions
- **HOD**: Use filters and analytics to efficiently manage department-wide requests
- **All Users**: Keep approval SMS messages for gate pass verification

## Security Notes
- Parent approval links are one-time use and expire after 24 hours
- All actions are logged with timestamps for accountability
- Only authorized users can access role-specific dashboards
- Approved requests are valid only until the specified leave time
